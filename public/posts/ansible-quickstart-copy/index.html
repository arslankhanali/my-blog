Ansible Quick Start on RHEL and macOS

August 4, 2025
#ansible
#automation
#rhel
#macos

A short and practical guide to getting started with Ansible on both RHEL and macOS. Includes installation, SSH setup, and a sample command.

Remote host example

Field Value
Username neo
Hostname node2
IP 192.168.50.205
OS Fedora

Install Ansible

# On RHEL
sudo dnf install -y ansible-core

# On macOS
brew install ansible

SSH Setup for Remote Host

# Check if you already have an SSH key
# Look for id*.pub file
ls ~/.ssh/

# If not, generate one
ssh-keygen -t rsa -b 4096

# Copy your public key to the remote host
ssh-copy-id -o StrictHostKeyChecking=no neo@192.168.50.205

Run an Ansible Playbook

Note the trailing comma , after the IP — it tells Ansible this is a direct host list and not an inventory file.

# Run with password prompt
ansible-playbook ping.yaml -u neo --ask-pass -i 192.168.50.205,

# Ping remote node
ansible all -i node1, -m ping -u neo

# Run shell command
ansible all -i node1, -m shell -a "uptime" -u neo

Next Steps

  • Use --ask-become-pass for privilege escalation
  • Use an inventory file for multiple nodes
  • Encrypt secrets with ansible-vault
  • Modularize using roles and collections

This setup gets you running with Ansible in under 5 minutes — ideal for homelab or quick automation testing.
